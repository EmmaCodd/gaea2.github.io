DOCTYPE与浏览器模式浅析

  • html

在群里看到的一个笔试题，“Doctype? 严格模式与混杂模式-如何触发这两种模式，区分它们有何意义?”

DOCTYPE概念

DOCTYPE，或者称为文档类型声明DTD。通常情况下，<!DOCTYPE>位于文档中的最前面的位置，处于标签之前。这样一来，在浏览器解析 HTML 文档正文之前就可以确定当前文档使用哪种 HTML 或 XHTML 规范。

如何触发这两种模式

如果XHTML文档包含形式完整的DOCTYPE，那么它一般以标准（严格）模式呈现。

DOCTYPE不存在或形式不正确会导致HTML和XHTML文档以混杂模式呈现。

这两种模式的表现差异

混杂模式没有兼容性可言。在IE6-IE9中，混杂模式都用IE5.5内核来解析并渲染页面。

标准模式要求标签必须闭合，不能使用已经废弃的标签等。

技术领先的公司（如谷歌、推特等）已经使用了html5的DOCTYPE声明，即<!DOCTYPE html>，好处是节省代码且向前兼容（HTML5时代）。
