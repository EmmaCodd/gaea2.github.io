grunt前端自动化

  • automation
  • grunt

从蒸汽时代开始，人类文明逐渐从手工向自动化靠拢。在计算机领域，对于需要反复重复的任务，比如压缩、编译、单元测试等，利用自动化工具可以节省精力，简化工作。本文将介绍如何使用grunt创建一个web项目，实现前端自动化。

安装grunt-CLI

前提是已安装node.js，如果没有，到https://nodejs.org 下载。在控制台输入以下代码测试node.js是否安装成功。

node -v
npm  -v

接下来在管理员权限下的命令行中运行 npm install -g grunt-cli ，这条命令将全局安装grunt命令。完成后测试grunt-cli看是否安装成功，在命令行输入grunt，如下图所示即可。

CLI 是如何工作的

每次运行grunt 时，它都利用node提供的require() 系统查找本地安装的 Grunt。由于这一机制，你可以在项目的任意子目录中运行grunt 。

如果找到一份本地安装的 Grunt，CLI就将其加载，并传递Gruntfile.js 文件中的配置信息，然后执行你所指定的任务。

在你的项目中添加两份文件：package.json 和 Gruntfile.js ，放置在项目根目录中。

安装grunt

npm install grunt --save-dev

安装完成后，devDependencies多了内容"grunt": "^1.0.1" ，grunt_test文件夹下多了一个名为”node_modules”的子文件夹。

此时在控制台输入grunt 命令，得到warning说明grunt在本目录下安装成功。
