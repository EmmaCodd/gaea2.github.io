跨域

  • jsonp
  • 跨域

JavaScript 出于安全的考虑，不允许跨域调用其他页面的对象。同时也给ajax 应用上带来了不少麻烦。

什么是跨域

一个域名下的网页,想访问另一个域名下的资源。跨域问题是由于js 的同源策略造成的。简单来说，同源策略指一段脚本只能读取来自同一来源的窗口和文档的属性，这里的同一来源指主机名、协议和端口号的组合。

URL 说明 是否允许通信
http://www.a.com/a.js
http://www.a.com/b.js		 同一域名下 允许
http://www.a.com/lab/a.js
http://www.a.com/script/b.js		 同一域名下不同文件夹 允许
http://www.a.com:8000/a.js
http://www.a.com/b.js		 同一域名，不同端口 不允许
http://www.a.com/a.js
https://www.a.com/b.js		 同一域名，不同协议 不允许
http://www.a.com/a.js
http://70.32.92.74/b.js		 域名和域名对应ip 不允许
http://www.a.com/a.js
http://script.a.com/b.js		 主域相同，子域不同 不允许
http://www.a.com/a.js
http://a.com/b.js		 同一域名，不同二级域名（同上） 不允许（cookie这种情况下也不允许访问）
http://www.cnblogs.com/a.js
http://www.a.com/b.js		 不同域名 不允许

禁止跨域请求

xhr (ajax 的请求是禁止跨域的)

如何解决(4种)

本文将介绍jsonp 跨域。

什么是jsonp

即填充式json。服务器端不仅只发送json 字符串，而是将json 字符串填充在一个js 语句中返回。

原理

利用动态创建的script 标签可以跨域，让服务器端返回可执行的javascript 函数，参数为要回发的数据。

如何(3种)

跨域方式一(原生js DOM)：
服务器端: 将原来直接发送的json 数据，填充到一个函数(例doit) 调用的js 语句中返回
客户端:
Step1: 预先定义一个函数,函数名(例doit) 要和服务器端js 语句的函数名一致
Step2: 创建script 标签, 请求服务器端php 返回的js 语句
