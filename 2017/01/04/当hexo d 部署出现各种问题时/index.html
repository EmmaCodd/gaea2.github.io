2017-01-04

当hexo d 部署出现各种问题时

  • git
  • hexo

fatal: could not read Username for https://github.com

前十分钟还顺利把hexo 本地同步到gayhub，下一秒git bash 满屏error…

看招不断，修改_config.yml 文件的deploy|repository，有点靠谱，但github 没SSH 时此举或无用。有说关掉localhost:4000再试的。改http为https再改为git@等等，都会引起错误类型的变化。玄，玄不救非。

我的做法是新建SSH key，再部署。

检查是否有SSH key

登陆github，点击头像Settings | SSH and GPG keys，看是否有SSH key。如果有，跳到第三步；若没有，继续。

新建 SSH key

在博客目录下打开git bash，输入命令$ ssh-keygen -t rsa -C "邮箱名"，双引号不可省。接下来

Generating public/private rsa key pair.
Enter file in which to save the key (某路径.ssh/id_rsa):

回车。若提醒某路径 .ssh/id_rsa already exists. Overwrite (y/n)? ，重写就行。

Enter passphrase (empty for no passphrase):
Enter same passphrase again:

输入自定义密码。

Your identification has been saved in 某路径.ssh/id_rsa.
Your public key has been saved in 某路径.ssh/id_rsa.pub.
The key fingerprint is:
SHA256:ygi*4 邮箱
The key’s randomart image is:
+[RSA 2048]-+
|        .O=+ .   |
| .       .  X     .  |
|o  X    X    +.. |
|. +.  X       o . |
|o .     S.     .+ |
|o .=oo. +.  S. |
| . + S.    . ..oo |
| +o  S.      ..+ |
| oE…  S.       . |
+–[SHA256]–+

在github上添加SSH key

Ⅰ、打开本地文件id_rsa.pub （文件路径可在git bash 回应代码中找到），复制全文。

Ⅱ、在github SSH keys | New SSH key。

Ⅲ、将Ⅰ 中复制的内容粘至Key，Title可缺省。

验证设置

$ ssh -T git@github.com

运行之前应已经把_config.yml 文件的deploy 中repository 替换为repo: git@github.com:username/username.github.io.git，注意第二个冒号。

出现

Hi username! You’ve successfully authenticated, but GitHub does not provide shell access.

设置用户信息

$ git config –global user.name “用户名”
$ git config –global user.email “邮箱”

部署

$ hexo g
$ hexo d

完成。
