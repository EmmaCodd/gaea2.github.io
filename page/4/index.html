gulp前端自动化

与grunt相比，gulp使用node.js串流(streams)使建构更快速，效率更高语法更自然,不需要编写复杂的配置文件。gulp利用来源档案当作输入，串流到一群外挂(plugins)，最后取得输出的结果，并非像grunt一样配置每一个外挂的输入与输出。

Installing Gulp

和grunt一样，它们都基于node.js，需要先搭建node环境。然后在命令行中cd到你的项目文件夹，在管理员权限下执行npm install -g gulp以全局安装gulp包。安装完成后，输入gulp -v查看gulp版本号验证gulp是否被正确安装。

DOCTYPE与浏览器模式浅析

在群里看到的一个笔试题，“Doctype? 严格模式与混杂模式-如何触发这两种模式，区分它们有何意义?”

DOCTYPE概念

DOCTYPE，或者称为文档类型声明DTD。通常情况下，<!DOCTYPE>位于文档中的最前面的位置，处于标签之前。这样一来，在浏览器解析 HTML 文档正文之前就可以确定当前文档使用哪种 HTML 或 XHTML 规范。

如何触发这两种模式

如果XHTML文档包含形式完整的DOCTYPE，那么它一般以标准（严格）模式呈现。

DOCTYPE不存在或形式不正确会导致HTML和XHTML文档以混杂模式呈现。
